Cloud Storage Cost Optimization

Google Cloud Storage offers features to optimize costs, such as Object Lifecycle Management, which automates actions like setting Time to Live (TTL) or transitioning objects to cheaper storage classes. Coldline and Archive Storage provide significant savings for infrequently accessed data, though with longer retrieval times. Additionally, Efficient Data Compression reduces storage space, helping organizations further cut costs while maintaining scalability

Apigee Spaces

Apigee Spaces enables identity-based isolation and grouping of API resources within an Apigee organization. Apigee Spaces allows you to have granular IAM control over access to your API proxies, shared flows, and API products. Developers can create multiple Spaces within the same organization for different teams, development projects, or environments and set IAM controls for each Space. Any API resources created under a Space will inherit the IAM policies applied to that Space.

Gemini Code Assist

Gemini Code Assist offers AI-powered assistance to help development teams build, deploy, and operate applications throughout the software development lifecycle. Developers can use Gemini Code Assist in their IDE (such as VS Code, JetBrains IDEs like IntelliJ and PyCharm, or Android Studio) for AI-powered coding assistance in many popular languages. It provides code completions as developers write their code, generates full functions or code blocks from comments, generates unit tests, and assists with debugging, understanding, and documenting code. Gemini Code Assist provides contextualized responses to prompts, including source citations regarding which documentation and code samples it used to generate its responses.

Blue Lab project - CICD pipeline for Apigee API Proxy deployment

In this proof of concept (POC), we aim to demonstrate the process of deploying an API proxy on Apigee using Cloud Build and a GitLab repository. The solution leverages a configuration JSON file to capture inputs, streamlining the deployment process. Additionally, an approval mechanism has been integrated, ensuring that the deployment is triggered only after the necessary build approval is granted. This POC not only enhances deployment efficiency but also adds an extra layer of control and security, maintaining high-quality standards throughout the process.

Security Updates
Critical Security Advisory
Important updates regarding Cloud Identity and Access Management. All teams must update IAM policies by March 15th to comply with new security standards.

Critical Security Advisory Details

All organizations using Cloud IAM must update their policies to include the new resource location constraints and session duration limits. These changes are required to meet updated ISO 27001 compliance standards. Failure to implement these changes by March 15th may result in service interruptions. Detailed migration guides and policy templates are available in the Security Hub.

Key changes include:

  • Mandatory multi-factor authentication for all admin roles
  • Session duration limits of 12 hours for privileged accounts
  • Geofencing requirements for sensitive data access
  • Enhanced audit logging configuration requirements
